# 📄 Test Case Document: Compare Products Feature

## TC-CP-003: Remove 1 product from the Compare page and verify the updated table

###  1. General Information
| Attribute | Details |
| :--- | :--- |
| **Test Case ID** | `TC-CP-003` |
| **Module** | Product Comparison |
| **Priority** | High |
| **Test Type** | Functional Testing |
| **Description** | Verify the workflow of removing a product from the comparison list, ensure the table UI updates correctly, and confirm that adding a new product afterward works as expected. |

---

###  2. Pre-conditions
* The user has already added 2 products to the compare list (**HTC Touch HD** and **Palm Treo Pro**).

---

###  3. Test Data
| Variable / Object | Input Value / Details |
| :--- | :--- |
| **Product 1** | Name: `HTC Touch HD` <br> Price: `$146.00` |
| **Product 2** | Name: `Palm Treo Pro` <br> Remove ID: `29` |
| **Product 3 (New)** | Name: `Nikon D300` <br> ID: `31` <br> Price: `$98.00` |
| **URL Parameter** | `?remove=29` |

---

###  4. Test Steps & Expected Results

| Step | Action | Expected Result |
| :---: | :--- | :--- |
| **1** | Add **HTC Touch HD** to compare using a JS click on the compare button. | A toast notification appears displaying: **"Product Compare (1)"**. |
| **2** | Add **Palm Treo Pro** to compare using a JS click on the compare button. | A toast notification appears displaying: **"Product Compare (2)"**. |
| **3** | Navigate to the URL: `/index.php?route=product/compare`. | The **Product Comparison** page loads, displaying the previously added products. |
| **4** | Verify that the table contains **2 products**. | The table correctly displays exactly 2 product columns. |
| **5** | Locate the **"Remove"** button for **Palm Treo Pro**. | The Remove button is present inside the Palm Treo Pro column. |
| **6** | Click the **"Remove"** button for **Palm Treo Pro**. | The page reloads successfully (URL briefly triggers `?remove=29`). |
| **7** | Verify the table UI after deletion. | Only **1 product** (HTC Touch HD) remains in the comparison table. |
| **8** | Confirm **Palm Treo Pro is no longer** displayed in the table. | The entire column for Palm Treo Pro is completely removed from the UI. |
| **9** | Verify **HTC Touch HD still remains** in the table. | The HTC Touch HD column is still displayed with all its original details. |
| **10** | Verify the price of the HTC Touch HD remains correct. | Displays Price: **$146.00**. |
| **11** | Verify the remaining number of Remove buttons. | There is exactly **1 Remove button** left on the page. |
| **12** | Verify the URL after the removal process. | The URL is exactly `route=product/compare` (the `remove` parameter is no longer present). |
| **13** | Add a new product, **Nikon D300** (ID: 31), to the compare list from the category page. | A toast notification appears displaying: **"Product Compare (2)"**. |
| **14** | Return to the Product Comparison page. | The table successfully displays both HTC Touch HD and Nikon D300. |
| **15** | Verify the **Nikon D300 Price** is correct. | Displays Price: **$98.00**. |

---

###  5. Expected Final Result
> After successfully removing the Palm Treo Pro, the table must update dynamically to contain only the HTC Touch HD with its correct price intact. Furthermore, the system must allow a new product (Nikon D300) to be added without any cache or session errors, correctly displaying the updated 2-product comparison table.
