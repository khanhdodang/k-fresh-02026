# 📄 Test Case Document: Compare Products Feature

## TC-CP-004: Remove all products — Verify the empty state

###  1. General Information
| Attribute | Details |
| :--- | :--- |
| **Test Case ID** | `TC-CP-004` |
| **Module** | Product Comparison |
| **Priority** | Medium |
| **Test Type** | Functional Testing |
| **Description** | Verify the workflow of removing all products from the comparison list, validate the empty state message, and ensure the "Continue" button functions correctly. |

---

###  2. Pre-conditions
* The user is currently on the Tablets category page.

---

###  3. Test Data
| Variable / Object | Input Value / Details |
| :--- | :--- |
| **Product 1** | Name: `HTC Touch HD` <br> ID: `28` |
| **Product 2** | Name: `Canon EOS 5D` <br> ID: `30` |

---

###  4. Test Steps & Expected Results

| Step | Action | Expected Result |
| :---: | :--- | :--- |
| **1** | Add **HTC Touch HD** (ID: 28) to compare. | A toast notification appears displaying: **"Product Compare (1)"**. |
| **2** | Add **Canon EOS 5D** (ID: 30) to compare. | A toast notification appears displaying: **"Product Compare (2)"**. |
| **3** | Navigate to the URL: `/index.php?route=product/compare`. | The comparison table is displayed. |
| **4** | Verify that the table correctly contains exactly 2 products. | Both HTC Touch HD and Canon EOS 5D are displayed in the table. |
| **5** | Click the **"Remove"** button for **HTC Touch HD** (ID: 28). | The page reloads successfully, leaving only 1 product in the table. |
| **6** | Verify that only **Canon EOS 5D** remains. | The table displays exactly 1 column named "Canon EOS 5D". |
| **7** | Verify that the page has no server errors. | The page loads completely without any 404 or 500 errors. |
| **8** | Click the **"Remove"** button for **Canon EOS 5D** (ID: 30). | The page reloads successfully. |
| **9** | Verify the page title. | The `H1` tag still correctly displays **"Product Comparison"**. |
| **10** | Verify the empty state message. | The page displays the text: **"You have not chosen any products to compare."** |
| **11** | Verify that no comparison table exists. | No `<table>` elements are rendered on the page. |
| **12** | Verify the presence of the **"Continue"** button/link. | The "Continue" button is clearly displayed. |
| **13** | Click the **"Continue"** button. | The system redirects the user. |
| **14** | Verify the URL after clicking Continue. | The user is redirected to the homepage or a valid previous category page. |
| **15** | Return to the compare page via direct URL (`/index.php?route=product/compare`). | The page still displays the empty state normally (no crashes or errors). |

---

###  5. Expected Final Result
> When all products are successfully removed from the comparison list, the system must handle the empty state gracefully by displaying the message *"You have not chosen any products to compare."* without any table structures or server errors. The *"Continue"* button must function correctly to navigate the user away from the empty state.
