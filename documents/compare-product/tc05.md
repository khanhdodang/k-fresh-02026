# 📄 Test Case Document: Compare Products Feature

## TC-CP-005: Add an existing product to Compare — Verify duplicate handling

###  1. General Information
| Attribute | Details |
| :--- | :--- |
| **Test Case ID** | `TC-CP-005` |
| **Module** | Product Comparison |
| **Priority** | Medium |
| **Test Type** | Functional Testing |
| **Description** | Verify the system's behavior when attempting to add a product that already exists in the comparison list, ensuring that no duplicate columns or erroneous data are generated. |

---

###  2. Pre-conditions
* The user is currently on the category page.
* The **Compare** list is empty (no previous products added).

---

###  3. Test Data
| Variable / Object | Input Value / Details |
| :--- | :--- |
| **Product 1** | Name: `Nikon D300` <br> ID: `31` <br> Price: `$98.00` |
| **Product 2** | Name: `iPod Touch` <br> ID: `32` <br> Price: `$194.00` |

---

###  4. Test Steps & Expected Results

| Step | Action | Expected Result |
| :---: | :--- | :--- |
| **1** | Navigate to the Tablets category page. | The page loads successfully, and the product list is displayed. |
| **2** | Click the **"Compare this Product"** button for **Nikon D300** (ID: 31) for the first time. | A toast notification appears displaying: *"You have added Nikon D300 to your product comparison!"* |
| **3** | Verify the first toast notification. | The link in the toast displays: **"Product Compare (1)"**. |
| **4** | Wait for the toast notification to disappear. | The toast notification is hidden successfully. |
| **5** | Click the **"Compare this Product"** button for **iPod Touch** (ID: 32). | A toast notification appears displaying: *"You have added iPod Touch to your product comparison!"* |
| **6** | Verify the second toast notification. | The link in the toast displays: **"Product Compare (2)"**. |
| **7** | Click the **"Compare this Product"** button for **Nikon D300** (ID: 31) for the second time. | The system processes the action without crashing. |
| **8** | Verify the toast notification content for the duplicate action. | The notification either warns that the product is *"already added"* OR shows success but keeps the comparison count unchanged at **"Product Compare (2)"**. |
| **9** | Navigate to the URL: `/index.php?route=product/compare`. | The **Product Comparison** page loads successfully. |
| **10** | Count the number of product columns in the table. | There are exactly **2 products** (Nikon D300 is not duplicated). |
| **11** | Verify the presence of Nikon D300. | Nikon D300 appears only **once** (no duplicate columns exist). |
| **12** | Verify the presence of iPod Touch. | iPod Touch is displayed in its own separate column. |
| **13** | Verify the price of **Nikon D300**. | Displays Price: **$98.00**. |
| **14** | Verify the price of **iPod Touch**. | Displays Price: **$194.00**. |
| **15** | Verify the total number of **Remove** buttons. | There are exactly **2 Remove buttons** corresponding to the two unique products (no extra buttons). |

---

###  5. Expected Final Result
> The system must handle duplicate additions gracefully. A product that already exists in the comparison list must not be added again, and the comparison table must strictly display unique items without any duplicated columns or distorted UI elements.
